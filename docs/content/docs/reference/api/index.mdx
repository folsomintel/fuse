The orchestrator exposes a REST API on [chi](https://github.com/go-chi/chi). Every
SDK and the CLI itself are built on this surface. The OpenAPI spec ships in the
repo at `internal/api/openapi.yaml` and covers the 18 `/v1/*` operations across
environments, snapshots, and hosts; the pages in this section additionally
document six real, live endpoints (`/v1/api-keys` and its two children, `/login`,
`/logout`, and `/metrics`) that exist in the router but aren't in the spec file,
sourced directly from the handler code.

## Base URL and path prefix

All resource routes are under `/v1`. A backward-incompatible change would bump
this prefix.

## Auth

Every `/v1/*` route requires a bearer token (or, for browser callers, the session
cookie); see [Authentication](/docs/reference/auth). `/login`, `/logout`,
`/health`, `/ready`, and `/metrics` are mounted outside auth entirely, the first
two so an unauthenticated browser can reach them, the last three for load
balancer/Kubernetes probes and Prometheus scraping.

## Request correlation

Every response carries an `X-Request-ID` header: either the value the client
supplied (if it matches `[A-Za-z0-9_-]{1,128}`) or a freshly generated
`req_<32hex>` identifier. The same value threads through audit events and server
logs. Probe endpoints (`/health`, `/ready`, `/metrics`) don't carry a request ID,
they're mounted outside the middleware that generates it.

## Error envelope

All non-2xx responses share one shape:

```json
{
  "error": {
    "code": "not_found",
    "message": "environment not found",
    "details": {}
  }
}
```

`code` is a stable, machine-readable string, one of:

| Code | Typical status |
| --- | --- |
| `not_found` | `404` |
| `conflict` | `409` |
| `invalid_argument` | `400` |
| `unauthorized` | `401` or `403` |
| `unavailable` | `503` |
| `unimplemented` | `501` |
| `internal` | `500` or `502` |

`details` is optional and free-form.
