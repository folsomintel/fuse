## Hosts

| Method | Path | Purpose |
| --- | --- | --- |
| `GET` | `/v1/hosts` | List hosts. |
| `POST` | `/v1/hosts` | Register a host. |
| `GET` | `/v1/hosts/{hostId}` | Fetch one host. |
| `POST` | `/v1/hosts/{hostId}?action=cordon` | Mark unschedulable. |
| `POST` | `/v1/hosts/{hostId}?action=uncordon` | Return to scheduling. |
| `DELETE` | `/v1/hosts/{hostId}` | Remove. `409` if VMs are still assigned. |

### Register request

```json
{
  "id": "prod-east-1",
  "url": "http://10.0.0.5:9000",
  "token": "...",
  "region": "us-east",
  "backend": "firecracker",
  "capacity": {
    "cpus": 16,
    "ram_mb": 65536,
    "storage_gb": 500,
    "vm_count": 20,
    "gpus": 0,
    "gpu_kind": ""
  }
}
```

`backend` defaults to `firecracker`. Only `qemu` hosts may register with
`capacity.gpus > 0`.

### Host states

`active`, `cordoned`, `draining`.

## API keys

| Method | Path | Purpose |
| --- | --- | --- |
| `POST` | `/v1/api-keys` | Create. Requires the master token. |
| `GET` | `/v1/api-keys` | List (metadata only). Requires the master token. |
| `DELETE` | `/v1/api-keys/{id}` | Revoke. Requires the master token. |

## Login / logout

Mounted outside the `BearerAuth` middleware group so an unauthenticated browser can
reach them:

| Method | Path | Purpose |
| --- | --- | --- |
| `POST` | `/login` | Body `{"token": "..."}`. On success, sets the HttpOnly `fuse_session` cookie. |
| `POST` | `/logout` | Clears the session cookie. Safe to call unauthenticated. |

See [Authentication](/docs/reference/auth) for the full session-cookie model and its
same-origin deployment constraint.
