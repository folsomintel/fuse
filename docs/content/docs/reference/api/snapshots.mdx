| Method | Path | Purpose |
| --- | --- | --- |
| `POST` | `/v1/environments/{vmId}/snapshots` | Snapshot a running VM. |
| `GET` | `/v1/snapshots` | List, with optional filters. |
| `GET` | `/v1/snapshots/{snapshotId}` | Fetch one. |
| `POST` | `/v1/snapshots/{snapshotId}?action=restore` | Restore in place onto the original VM. |
| `DELETE` | `/v1/snapshots/{snapshotId}` | Delete (also removes the provider artifact). |

See [Snapshots](/docs/concepts/snapshots) for the concept-level model,
including the QEMU/GPU exception (no snapshot support at all).

## Create

```
POST /v1/environments/{vmId}/snapshots
```

### Request body (optional)

```json
{
  "comment": "before risky migration",
  "mode": "manual",
  "retention_seconds": 0,
  "metadata": { "key": "value" },
  "export_ref": "",
  "export_status": "pending"
}
```

`vmId` must be a `running` environment. `mode` is `manual` or `auto`.
`retention_seconds` of `0` keeps the snapshot forever. The whole body is
optional, an empty body is a valid request.

### Response

`201` with the created `Snapshot`:

```json
{
  "id": "snap-1",
  "vm_id": "fuse-abc123",
  "task_id": "t-1",
  "mode": "manual",
  "state": "creating",
  "comment": "before risky migration",
  "size_bytes": 0,
  "created_at": "2026-01-15T10:00:00Z"
}
```

### Errors

| Status | Code | When |
| --- | --- | --- |
| `404` | `not_found` | `vmId` doesn't exist. |
| `500` | `internal` | The VM isn't `running`, the environment doesn't support snapshots (GPU, or any QEMU-backed VM), or an unexpected server error. See the note below. |

:::warning[Known limitation]
Two permanent request conditions currently return `500` `internal` rather than
a `4xx`. A snapshot of a VM that isn't `running` reports `vm ... in state ...:
snapshots require running`. An unsupported-backend snapshot reports `vm ... has
a gpu passthrough device: snapshots are not supported for gpu environments` for
a GPU environment, and `provider does not support snapshots for vm ...` for any
other non-snapshotting backend. Each is a permanent property of the request
rather than a server fault, so the status should be a `4xx`; match on the
message rather than the status until this is fixed.
:::

## List

```
GET /v1/snapshots?vm_id=&task_id=&tenant_id=&state=
```

All four query parameters are optional exact-match filters. Returns `200`
with `{"snapshots": [...]}`. There is no host-scoped filter, snapshot
listing is fleet-wide regardless of which host the source VM was on.

## Get

```
GET /v1/snapshots/{snapshotId}
```

Returns `200` with the full `Snapshot`:

```json
{
  "id": "snap-1",
  "vm_id": "fuse-abc123",
  "task_id": "t-1",
  "tenant_id": "",
  "parent_snapshot_id": "",
  "mode": "manual",
  "state": "ready",
  "comment": "before risky migration",
  "size_bytes": 1288490188,
  "created_at": "2026-01-15T10:00:00Z",
  "updated_at": "2026-01-15T10:01:00Z",
  "retention_until": null,
  "last_error": "",
  "export_ref": "",
  "exports": []
}
```

`state` is one of `creating`, `ready`, `restoring`, `deleting`, or `error`.
`retention_until` is `null` when the snapshot is kept forever. Or `404`
`not_found` if it doesn't exist.

## Restore

```
POST /v1/snapshots/{snapshotId}?action=restore
```

Restores the snapshot **in place**, onto its original VM, which must still be
running. There is no cross-host restore, clone, or standalone restore into a
new VM, use the fork action on
[Environments](/docs/reference/api/environments#fork) for that instead.
Returns `204` on success.

:::warning[Known limitation]
Restoring while the guest has unsynced writes can corrupt the restored data.
See
[Troubleshooting](/docs/cloud/troubleshooting#snapshot-restore-can-corrupt-unsynced-writes)
for the workaround.
:::

## Delete

```
DELETE /v1/snapshots/{snapshotId}
```

Leaf-only: also removes the underlying provider artifact. There is no
soft-delete. Returns `204` on success, `404` `not_found` if it doesn't exist,
or `409` `conflict` if another snapshot lists this one as its parent, which is
the lineage record a fork writes. Exports are not consulted and do not block
deletion.
