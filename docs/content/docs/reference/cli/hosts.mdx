## List

```bash
fuse hosts              # or: fuse hosts list / fuse hosts ls
```

## Select an active host

```bash
fuse host <id>     # select
fuse host          # show current selection
```

Scopes host-relative commands (like `fuse environment list`'s default filter) to
this host. Does not pin scheduling, see [Environments](/docs/reference/cli/environments).

## Register

```bash
fuse host register <id> --url <url> [flags]
```

| Flag | Purpose |
| --- | --- |
| `--url` | Host agent base URL. Required. |
| `--region` | Region label. |
| `--token` | Token the orchestrator uses to call this host's agent. |
| `--backend` | `firecracker` (default) or `qemu`. |
| `--cpus`, `--ram-mb`, `--storage-gb`, `--max-vms` | Capacity. |
| `--gpus` | GPU device count. Requires `--backend qemu`. |
| `--gpu-kind` | GPU model label, e.g. `a100`. |

Omit `--url` in an interactive terminal to fill in capacity fields via a form
instead.

## Get, cordon, uncordon, remove, metrics

```bash
fuse host get <id>
fuse host cordon <id>
fuse host uncordon <id>
fuse host remove <id> [--yes]     # aliases: rm, deregister; refuses if VMs assigned
fuse host metrics [id]            # defaults to the active host
```

`fuse host metrics` shows capacity vs. allocated (CPUs, RAM, storage, VM count, and
GPUs where applicable), there is no per-VM metrics API, only per-host.
