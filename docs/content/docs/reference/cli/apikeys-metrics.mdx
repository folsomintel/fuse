## API keys

```bash
fuse apikeys create [--label "..."]   # aliases: apikey, keys
fuse apikeys list                      # alias: ls
fuse apikeys revoke <id>
```

All `apikeys` commands require the master token, the active context must be
connected with `--master` (see [Connecting and contexts](/docs/reference/cli/contexts)).
`create` prints the raw secret once; it cannot be recovered afterward. `list` shows
metadata only (id, label, created, last used, revoked).

## Metrics

```bash
fuse metrics
```

Scrapes the orchestrator's `/metrics` endpoint and prints the raw Prometheus
exposition (fleet-wide aggregates). There is no per-VM metrics API, for per-host
capacity use `fuse host metrics <id>` (see [Hosts](/docs/reference/cli/hosts)).
