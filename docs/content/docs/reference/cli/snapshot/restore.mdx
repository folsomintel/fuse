---
title: fuse snapshot restore
description: Restore a snapshot in place
icon: RotateCcw
---

```bash
fuse snapshot restore <id> [--yes]
```

No aliases. Restores the snapshot **in place**: onto its original VM, which
must still be running. There is no cross-host restore, clone, or standalone
restore into a new VM, use
[`fuse environment fork`](/docs/reference/cli/environment/fork) for that
instead.

## Flags

| Flag | Purpose |
| --- | --- |
| `--yes` | Skip the confirmation prompt. |

<Callout type="warn" title="Known limitation">
Restoring while the guest has unsynced writes can corrupt the restored data.
See
[Troubleshooting](/docs/cloud/troubleshooting#snapshot-restore-can-corrupt-unsynced-writes)
for the workaround.
</Callout>

Without `--yes` in an interactive terminal, prompts for confirmation and warns
that this overwrites the running VM.
