---
title: fuse up
description: Compile a Fusefile and create an environment from it
icon: rocket
---

```bash
fuse up [path] [flags]
```

Reads a [Fusefile](/docs/concepts/fusefile), compiles it into a resource
spec, manifest, and startup script, then creates an environment from the
result. Streams provisioning events until the environment settles, unless
`--no-wait` is set.

`up` returns as soon as the environment reaches `running`, so it is safe to
use in scripts and CI. If the environment reaches `failed` or is destroyed
before it comes up, `up` exits non-zero. To keep watching an environment after
it is up, use [`fuse environment watch`](/docs/reference/cli/environment/watch),
which streams until the environment is destroyed or fails.

## Flags

| Flag | Purpose |
| --- | --- |
| `-f, --file` | Path to the Fusefile. Default: `./Fusefile`, or a positional path argument. |
| `--secret` | Secret as `key=value`. Repeatable. Overrides `--secrets-file` on key collision. |
| `--secrets-file` | Path to a file of `KEY=VALUE` secret lines. `#`-prefixed lines and blanks are ignored. |
| `--task-id` | Environment task ID. Default: the Fusefile's parent directory name. |
| `--no-wait` | Create the environment without streaming provisioning events. |

## Path resolution

The Fusefile path is resolved in this priority order: `-f/--file` if set, then
a positional argument, then `./Fusefile` as the default. Both `-f` and the
positional argument name the **file** itself, neither appends `Fusefile` to a
directory, so pointing either at a directory fails with
`read ./services/api: read ./services/api: is a directory` (the operation and
path appear twice because the CLI prefixes an error that already names them).

```bash
fuse up                                 # ./Fusefile
fuse up ./services/api/Fusefile
fuse up -f custom.fusefile.yaml
```

## Secrets

```bash
fuse up --secret pg_password=hunter2
fuse up --secrets-file .env.secrets
fuse up --secrets-file .env.secrets --secret pg_password=override  # override wins on collision
```

If any secret named in the Fusefile's `secrets` list is missing from the
resolved set (flags plus file), `up` fails fast with the list of missing names
rather than creating a broken environment.

## Examples

```bash
fuse up
# creating environment fuse-abc123 (task my-service)
# provisioning
# running
```

```bash
fuse up --no-wait
# creates immediately, does not stream events; watch separately with
# `fuse environment watch <id>`
```

See [Fusefile](/docs/concepts/fusefile) for the full field reference and
[`fuse init`](/docs/reference/cli/init) to scaffold one.
