## Connect

```bash
fuse connect <url> [--token <token>] [--name <name>] [--master]
```

Stores an orchestrator's base URL and bearer token as a named context and makes it
active. The URL is the orchestrator root (`scheme://host:port`), without a `/v1`
suffix.

| Flag | Purpose |
| --- | --- |
| `--token` | Bearer token for the orchestrator. |
| `--name` | Context name (default: derived from the URL's hostname). |
| `--master` | Mark this token as the master token, required for `fuse apikeys` commands. Informational client-side; the server is the source of truth on whether the token actually has master privileges. |

With no `--token`, connecting only works against an orchestrator running in
insecure/dev mode (no `ORCH_AUTH_TOKEN` set); the CLI warns you when you do this.

## Managing contexts

```bash
fuse context list              # aliases: contexts, ctx
fuse context use <name>
fuse context remove <name>     # alias: rm
fuse context current
```

`fuse context list` never prints the stored token, only name, base URL, whether
it's marked master, active host, and which one is current.
