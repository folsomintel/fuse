`client.apiKeys` is an `ApiKeysService`. All three methods require the master
token, the server enforces this regardless of what token the client was
constructed with.

## create

```ts
create(label?: string, opts?: CallOptions): Promise<CreatedAPIKey>
```

```ts
interface APIKey {
  id: string;
  label?: string;
  created_at: string;
  last_used_at?: string;
  revoked_at?: string;
}

interface CreatedAPIKey extends APIKey {
  key: string;
}
```

`label` is an optional human-readable name. `CreatedAPIKey.key` holds the raw
secret and is returned exactly once, it cannot be recovered afterward. Store it
immediately.

```ts
const key = await client.apiKeys.create("ci-runner");
console.log("save this now:", key.key);
```

## list

```ts
list(opts?: CallOptions): Promise<APIKey[]>
```

Returns metadata only, `APIKey` has no `key` field. `last_used_at` and
`revoked_at` are absent until they happen.

## revoke

```ts
revoke(id: string, opts?: CallOptions): Promise<void>
```

Revokes the key with the given `id` (the `APIKey.id`, not the secret itself).
