`client.apiKeys` mints and revokes API keys. All three methods require the
master token, the server enforces this regardless of what token the client
was constructed with.

```ts
const key = await client.apiKeys.create("ci-runner");
console.log("save this now:", key.key); // shown exactly once
```

`label` is optional. `key.key` holds the raw secret and is returned exactly
once, it cannot be recovered afterward, store it immediately.

## Everything else

| Method | Purpose |
| --- | --- |
| `list()` | List key metadata, `APIKey` has no `key` field. `last_used_at`/`revoked_at` are absent until they happen. |
| `revoke(id)` | Revoke the key with the given `id` (the `APIKey.id`, not the secret itself). |

These mirror the [HTTP API](/docs/reference/api/api-keys) one to one.
