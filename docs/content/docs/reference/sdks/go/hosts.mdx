`client.Hosts` is a `*HostsService`. It registers and manages the compute hosts the
scheduler places microVMs onto.

## Register

```go
func (s *HostsService) Register(ctx context.Context, req RegisterHostRequest) (*Host, error)
```

```go
type RegisterHostRequest struct {
	ID       string
	URL      string
	Token    string
	Region   string
	Backend  string
	Capacity HostCapacity
}

type HostCapacity struct {
	CPUs      int
	RamMB     int
	StorageGB int
	VMCount   int
	GPUs      int
	GPUKind   string
}
```

`ID` is a free-form, operator-supplied string and doubles as the host's primary
key. `Backend` defaults to `"firecracker"` server-side if left empty; only
`"qemu"` hosts may register with `Capacity.GPUs > 0`.

`Capacity.CPUs`, `RamMB`, and `StorageGB` are optional hardware facts: leave any
of them at `0` and the orchestrator probes the host agent for the real value
instead of trusting a guess. Pass one explicitly to override the probe (for a
deliberate overcommit or carve-out); a declared value above what the probe
reports still registers, with a warning rather than a rejection. `VMCount` is a
scheduling policy, not something a host can report on its own, so it is never
probed and must always be set.

```go
host, err := client.Hosts.Register(ctx, fuse.RegisterHostRequest{
	ID:      "prod-east-1",
	URL:     "http://10.0.0.5:9000",
	Token:   agentToken,
	Region:  "us-east",
	Backend: "firecracker",
	Capacity: fuse.HostCapacity{
		VMCount: 20, // required; cpus/ram_mb/storage_gb left at 0 to probe
	},
})
for _, w := range host.Warnings {
	log.Println("warning:", w)
}
```

If the host agent's provider doesn't support capacity probing and any of
`CPUs`/`RamMB`/`StorageGB` were left at `0`, registration fails with a `502`
`*APIError`, declare them explicitly in that case.

## List

```go
func (s *HostsService) List(ctx context.Context) ([]Host, error)
```

Returns every registered host, each showing its declared `Capacity` and live
`Allocated` figures.

## Get

```go
func (s *HostsService) Get(ctx context.Context, hostID string) (*Host, error)
```

```go
type Host struct {
	ID        string
	URL       string
	Region    string
	State     string
	Backend   string
	Capacity  HostCapacity
	Allocated HostCapacity
	LastSeen  time.Time
	CreatedAt time.Time
	UpdatedAt time.Time
	Warnings  []string
}
```

`State` is one of `active`, `cordoned`, or `draining`. `LastSeen` is set at
registration and is not refreshed afterward, it is not a liveness signal.
`Warnings` is only ever populated on the response to `Register`, non-fatal
notices such as a declared capacity value that exceeded what was probed from
the host agent.

## Cordon and Uncordon

```go
func (s *HostsService) Cordon(ctx context.Context, hostID string) error
func (s *HostsService) Uncordon(ctx context.Context, hostID string) error
```

Cordoning marks a host unschedulable: the scheduler stops placing new
environments there without touching what's already running. Uncordoning reverses
it.

## Deregister

```go
func (s *HostsService) Deregister(ctx context.Context, hostID string) error
```

Removes a host from the scheduler. Fails with a `conflict` `*APIError` if any
environments are still assigned to it, drain or destroy them first.
