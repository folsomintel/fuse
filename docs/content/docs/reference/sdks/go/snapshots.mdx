`client.Snapshots` is a `*SnapshotsService`. It captures and restores the state of
a running microVM.

## Create

```go
func (s *SnapshotsService) Create(ctx context.Context, vmID string, reqBody SnapshotRequest) (*Snapshot, error)
```

```go
type SnapshotRequest struct {
	Comment          string
	Mode             string
	RetentionSeconds int64
	Metadata         map[string]string
	ExportRef        string
	ExportStatus     string
}
```

`vmID` must be a `running` environment. `Mode` is `"manual"` or `"auto"`.
`RetentionSeconds` of `0` keeps the snapshot forever. All fields on
`SnapshotRequest` are optional, a zero-value struct is a valid request.

```go
snap, err := client.Snapshots.Create(ctx, env.ID, fuse.SnapshotRequest{
	Comment: "before risky migration",
	Mode:    "manual",
})
```


```go
type ListSnapshotsOptions struct {
	VMID     string

	State    string
}
```

All four filters are optional exact matches.

## Get

```go
func (s *SnapshotsService) Get(ctx context.Context, snapshotID string) (*Snapshot, error)
```

```go
type Snapshot struct {
	ID               string
	VMID             string
	TaskID           string
	TenantID         string
	ParentSnapshotID string
	Mode             string
	State            string
	Comment          string
	SizeBytes        int64
	CreatedAt        time.Time
	UpdatedAt        time.Time
	RetentionUntil   *time.Time
	LastError        string
	ExportRef        string
	Exports          []SnapshotExport
}
```

`State` is one of `creating`, `ready`, `restoring`, `deleting`, or `error`.
`RetentionUntil` is `nil` when the snapshot is kept forever.

## Delete

```go
func (s *SnapshotsService) Delete(ctx context.Context, snapshotID string) error
```

Leaf-only: also deletes the underlying provider artifact. There is no soft-delete.

## Restore

```go
func (s *SnapshotsService) Restore(ctx context.Context, snapshotID string) error
```

Restores the snapshot in place, onto its original VM, which must still be running.
There is no cross-host restore, clone, or standalone restore into a new VM, use
`Fork` on `EnvironmentsService` for that instead.

<Callout type="warn" title="Known limitation">
Restoring while the guest has unsynced writes can corrupt the restored data. See
[Troubleshooting](/docs/cloud/troubleshooting#snapshot-restore-can-corrupt-unsynced-writes)
for the workaround.
</Callout>

## The QEMU/GPU exception

Environments running on the QEMU backend (whole-GPU passthrough) cannot be
snapshotted or forked at all, `Create` and `Restore` against a GPU environment
fail with a `conflict` `*APIError` rather than silently no-op-ing. See
[Providers](/docs/learn/concepts/providers).
