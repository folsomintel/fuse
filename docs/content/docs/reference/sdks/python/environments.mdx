`client.environments` is an `EnvironmentsService`. It provisions, inspects, and
tears down microVMs.

## list

```python
def list(
    self, *, task_id: str = "", state: str = "", host_id: str = ""
) -> list[EnvironmentInfo]: ...
```

All three filters are optional exact matches; call with no arguments to list
everything.

```python
envs = client.environments.list(state=fuse.STATE_RUNNING)
```

## get

```python
def get(self, vm_id: str) -> EnvironmentInfo: ...
```

Fetches one environment by its assigned ID. Raises `fuse.ApiError` with
`code == "not_found"` if it doesn't exist.

## create

```python
def create(self, request: CreateRequest) -> EnvironmentInfo: ...
```

```python
class CreateRequest(BaseModel):
    task_id: str
    spec: Spec = Spec()
    manifest_inline: str | None = None
    secrets: dict[str, str] | None = None
    startup_script: str | None = None
    gateway_url: str | None = None
    gateway_token: str | None = None
    expose: list[ExposeSpec] | None = None

class Spec(BaseModel):
    cpus: int | None = None
    ram_mb: int | None = None
    storage_gb: int | None = None
    gpus: int | None = None
    gpu_kind: str | None = None
    region: str | None = None
    max_runtime_seconds: int | None = None
    image: str | None = None
```

`task_id` must be unique. `gpus` is a whole-device count; a non-zero value routes
the environment to a host registered with matching free GPU capacity (necessarily
a QEMU-backed host). `create` blocks server-side until the environment reaches
`running` or fails, so the returned value already reflects the outcome.

```python
env = client.environments.create(
    fuse.CreateRequest(
        task_id="task-1",
        spec=fuse.Spec(cpus=2, ram_mb=2048, storage_gb=10),
        secrets={"pg_password": "..."},
    )
)
```

## drain

```python
def drain(self, vm_id: str) -> EnvironmentInfo: ...
```

Phase one of a two-phase teardown: signals the guest agent to stop gracefully. The
environment moves to `draining` but is not destroyed; call `destroy` afterward to
remove it. Only `running` environments can be drained; any other state raises
`fuse.ApiError` with `code == "conflict"`.

## fork

```python
def fork(self, vm_id: str, options: ForkOptions | None = None) -> EnvironmentInfo: ...
```

```python
class ForkOptions(BaseModel):
    reuse_snapshot_id: str | None = None
    comment: str | None = None
```

Creates a new environment seeded from an existing one's state. With
`reuse_snapshot_id` unset, the server takes a fresh snapshot of `vm_id` first;
otherwise it reuses the named snapshot. Fork is unavailable wherever snapshotting
is, GPU/QEMU-backed environments cannot be forked.

## rotate_token

```python
def rotate_token(self, vm_id: str) -> None: ...
```

Re-issues the guest agent's credentials without recreating the VM. Returns
nothing on success (the server responds `204`).

## destroy

```python
def destroy(self, vm_id: str) -> None: ...
```

Forcefully tears the environment down from any state. Idempotent: a second call
against an already-destroyed environment raises `fuse.ApiError` with
`code == "not_found"`, not a different failure mode.

## events

```python
def events(self, vm_id: str) -> Iterator[Event]: ...
```

```python
class Event(BaseModel):
    id: str = ""
    kind: str = ""
    vm_id: str = ""
    state: str = ""
    url: str = ""
    error: str = ""
    updated_at: datetime | None = None
    err: Exception | None = None
```

Opens the SSE event stream and returns an iterator of `Event`. The first item is a
snapshot of the current state, yielded immediately on connect; subsequent items
fire on every transition. Iteration ends cleanly on EOF, after a terminal-state
event, or after a final `Event` with `.err` set if the underlying stream failed,
check `event.err` on every iteration.

```python
for event in client.environments.events(env.id):
    if event.err:
        raise event.err
    if event.state == fuse.STATE_RUNNING:
        break
```

## Lifecycle states

```python
STATE_PROVISIONING = "provisioning"
STATE_RUNNING = "running"
STATE_DRAINING = "draining"
STATE_DESTROYING = "destroying"
STATE_DESTROYED = "destroyed"
STATE_FAILED = "failed"

def is_terminal_state(state: str) -> bool: ...
```

`is_terminal_state` reports whether `state` is `destroyed` or `failed`, the two
states after which an event stream ends.
