`client.hosts` is a `HostsService`. It registers and manages the compute hosts the
scheduler places microVMs onto.

## register

```python
def register(self, request: RegisterHostRequest) -> Host: ...
```

```python
class RegisterHostRequest(BaseModel):
    id: str
    url: str
    token: str | None = None
    region: str | None = None
    backend: str | None = None
    capacity: HostCapacity = HostCapacity()

class HostCapacity(BaseModel):
    cpus: int = 0
    ram_mb: int = 0
    storage_gb: int = 0
    vm_count: int = 0
    gpus: int = 0
    gpu_kind: str = ""
```

`id` is a free-form, operator-supplied string and doubles as the host's primary
key. `backend` defaults to `"firecracker"` server-side if left unset; only
`"qemu"` hosts may register with `capacity.gpus > 0`.

`capacity.cpus`, `ram_mb`, and `storage_gb` are optional hardware facts: leave
any at `0` and the orchestrator probes the host agent for the real value
instead of trusting a guess. Pass one explicitly to override the probe (for a
deliberate overcommit or carve-out); a declared value above what the probe
reports still registers, with a warning rather than a rejection. `vm_count` is a
scheduling policy, not something a host can report on its own, so it is never
probed and must always be set.

```python
host = client.hosts.register(
    fuse.RegisterHostRequest(
        id="prod-east-1",
        url="http://10.0.0.5:9000",
        token=agent_token,
        region="us-east",
        backend="firecracker",
        capacity=fuse.HostCapacity(vm_count=20),  # cpus/ram_mb/storage_gb left at 0 to probe
    )
)
for w in host.warnings:
    print("warning:", w)
```

If the host agent's provider doesn't support capacity probing and any of
`cpus`/`ram_mb`/`storage_gb` were left at `0`, registration raises
`fuse.ApiError` with `status == 502`, declare them explicitly in that case.

## list

```python
def list(self) -> list[Host]: ...
```

Returns every registered host, each showing its declared `capacity` and live
`allocated` figures.

## get

```python
def get(self, host_id: str) -> Host: ...
```

```python
class Host(BaseModel):
    id: str = ""
    url: str = ""
    region: str = ""
    state: str = ""
    backend: str = ""
    capacity: HostCapacity = HostCapacity()
    allocated: HostCapacity = HostCapacity()
    last_seen: datetime | None = None
    created_at: datetime | None = None
    updated_at: datetime | None = None
    warnings: list[str] = []
```

`state` is one of `active`, `cordoned`, or `draining`. `last_seen` is set at
registration and is not refreshed afterward, it is not a liveness signal.
`warnings` is only ever populated on the response to `register`, non-fatal
notices such as a declared capacity value that exceeded what was probed from
the host agent.

## cordon and uncordon

```python
def cordon(self, host_id: str) -> None: ...
def uncordon(self, host_id: str) -> None: ...
```

Cordoning marks a host unschedulable: the scheduler stops placing new
environments there without touching what's already running. Uncordoning reverses
it.

## deregister

```python
def deregister(self, host_id: str) -> None: ...
```

Removes a host from the scheduler. Raises `fuse.ApiError` with
`code == "conflict"` if any environments are still assigned to it, drain or
destroy them first.
