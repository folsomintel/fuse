```bash
fuse environment destroy <id> [--yes]
```

Aliases: `rm`, `delete`. Forcefully tears the environment down from any state.

## Flags

| Flag | Purpose |
| --- | --- |
| `--yes` | Skip the confirmation prompt. |

## Behavior

Destroy is **not** idempotent: destroying an environment that no longer exists
returns a not-found error and a non-zero exit code, so scripts that tolerate a
missing environment have to check first or ignore the exit code.

Without `--yes` in an interactive terminal, prompts for confirmation first.
Outside a terminal there is no prompt to answer, so destroy aborts with
`aborted (use --yes to skip confirmation)` rather than proceeding. Always pass
`--yes` in scripts and CI.

To tear down gracefully instead of forcefully, drain first:

```bash
fuse environment drain <id>
fuse environment destroy <id>
```

See [`fuse environment drain`](/reference/cli/environment/drain).
