`fuse` is the operator CLI for the Fuse orchestrator, built with Cobra. It lives in
the repo's `cli/` package.

## Install

```bash
go build -o bin/fuse ./cli
```

## Global flags

These apply to every command:

| Flag | Purpose |
| --- | --- |
| `-o, --output` | Output format: `table` (default) or `json`. |
| `--context` | Context to use for this invocation, overriding the active context. |
| `--config` | Path to the config file (default: `$XDG_CONFIG_HOME/fuse/config.yaml`, or `~/.config/fuse/config.yaml`). |

## Config file

CLI state, orchestrator contexts (base URL + token) and each context's active host
selection, persists to a YAML file:

```
$XDG_CONFIG_HOME/fuse/config.yaml     # or ~/.config/fuse/config.yaml
```

The file is written with `0600` permissions (it holds bearer tokens) inside a `0700`
directory, created automatically on first `fuse connect`.

## Command groups

| Command | Purpose |
| --- | --- |
| `fuse quickstart` | Guided first-time setup: connect an orchestrator and register a host. |
| `fuse connect` / `fuse context` | Add and switch between orchestrator contexts. See [Contexts](/reference/cli/contexts). |
| `fuse hosts` / `fuse host` | Manage compute hosts. See [Hosts](/reference/cli/hosts). |
| `fuse environment` | Manage environments (VMs) directly. See [Environments](/reference/cli/environment). |
| `fuse up` | Compile a [Fusefile](/docs/concepts/fusefile) and create an environment from it. See [`fuse up`](/reference/cli/up). |
| `fuse init` | Scaffold an example Fusefile. See [`fuse init`](/reference/cli/init). |
| `fuse snapshot` | Manage VM snapshots. See [Snapshots](/reference/cli/snapshot). |
| `fuse apikeys` | Manage orchestrator-global API keys. See [API keys](/reference/cli/apikeys). |
| `fuse metrics` | Print fleet-wide Prometheus metrics. See [`fuse metrics`](/reference/cli/metrics). |

Most commands print human-readable tables by default; pass `-o json` for
machine-readable output, which is the same shape returned by the [HTTP API](/reference/api).
