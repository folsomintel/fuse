This section is the reference for driving a running Fuse orchestrator: the three
client SDKs, the operator CLI, and the raw HTTP API underneath both. If you're new
to Fuse's concepts, start with [Concepts](/docs/concepts/fusefile) first. This section assumes you
already know what an environment, host, and snapshot are.

## Picking a surface

<CardGroup cols={2}>
  <Card title="CLI" href="/reference/cli" icon="terminal">The fastest way to operate Fuse by hand: register hosts, bring up environments, inspect and tear things down.</Card>
  <Card title="SDKs" href="/reference/go" icon="braces">Drive Fuse programmatically from your own service or agent harness. Go, Python, and TypeScript, all mirroring the same shape.</Card>
  <Card title="HTTP API" href="/reference/api" icon="webhook">The raw REST surface every SDK and the CLI itself are built on.</Card>
  <Card title="Authentication" href="/reference/auth" icon="key-square">How the orchestrator authenticates callers, across the CLI, SDKs, and browser.</Card>
</CardGroup>

## Resource coverage

Every surface exposes the same four resources:

| Resource | What it manages |
| --- | --- |
| Environments | Create, list, inspect, drain, fork, destroy microVMs. |
| Hosts | Register, list, cordon/uncordon, remove compute hosts. |
| Snapshots | Create, list, restore, delete VM snapshots. |
| API keys | Issue and revoke orchestrator-global API keys (master-token only). |
